# Принос към OMAYA Platform

Благодарим ви, че проявихте интерес към приноса към OMAYA Platform! Този документ съдържа насоки за това как да допринесете за проекта.

## Технологичен стек

Проектът използва:
- **Frontend**: React (Vite, TypeScript), Tailwind CSS, Radix UI.
- **Backend**: FastAPI (Python), TimescaleDB, Redis, Kafka.
- **Edge**: Rust (за високопроизводителни конектори).
- **Инфраструктура**: Docker, Kubernetes, Helm.

## Как да допринесете

### Докладване на бъгове

Ако откриете бъг, моля, отворете Issue в GitHub, като използвате съответния шаблон. Опишете стъпките за възпроизвеждане и очакваното поведение.

### Предлагане на нови функционалности

Отворете Issue с описание на новата функционалност и защо тя е полезна за проекта.

### Pull Requests

1. Форкнете хранилището.
2. Създайте нов branch (`git checkout -b feature/amazing-feature`).
3. Направете вашите промени.
4. Уверете се, че всички тестове минават (`npm test` за фронтенд, `pytest` за бекенд).
5. Направете commit на вашите промени с ясно съобщение.
6. Направете push към вашия branch.
7. Отворете Pull Request.

## Стандарти за код

### Frontend (TypeScript/React)
- Използвайте ESLint и Prettier за форматиране.
- Предпочитайте функционални компоненти и Hooks.
- Всички нови компоненти трябва да имат съответните юнит тестове в `vitest`.
- Използвайте Zod за валидация на схеми.

### Backend (Python/FastAPI)
- Следвайте PEP 8 и използвайте `black` за форматиране.
- Използвайте Pydantic модели за валидация на данни.
- Всички нови API ендпоинти трябва да включват OpenAPI документация (docstrings).
- Тествайте с `pytest`.

### Edge (Rust)
- Rust кодът трябва да бъде форматиран с `rustfmt`.
- Използвайте `clippy` за статичен анализ.
- Осигурете пълна обработка на грешки без `panic!`.
- Документирайте публичните API-та.

## Език

Документацията и комуникацията по този проект се водят приоритетно на български език, но програмният код и коментарите в него трябва да бъдат на английски език.
