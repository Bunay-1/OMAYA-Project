# ADR 001: Избор на Rust за Edge конектори

## Статус
Прието

## Контекст
Платформата OMAYA изисква висока производителност и надеждност при комуникацията с индустриално оборудване (PLC, CNC). Edge конекторите трябва да обработват голям обем данни в реално време с минимална латентност и ниска консумация на ресурси.

## Решение
Решихме да използваме Rust за имплементация на критичните Edge конектори (Modbus, MQTT Sparkplug B, OPC-UA).

## Аргументация
1. **Производителност**: Rust предлага скорост, близка до C/C++, което е критично за industrial-grade приложения.
2. **Безопасност на паметта**: Моделът на собственост (ownership) на Rust елиминира цели класове от бъгове, свързани с паметта (memory leaks, segmentation faults).
3. **Concurrency**: Rust позволява безопасно паралелно изпълнение, което е важно за едновременна комуникация с множество машини.
4. **Binary Size**: Генерираните бинарни файлове са малки и не изискват тежки runtime среди, което ги прави идеални за edge устройства.

## Последствия
- Увеличава се сложността на технологичния стек (изисква се познаване на Rust).
- Необходимост от допълнителни CI/CD стъпки за компилиране на Rust кода.
- Значително подобрена стабилност и производителност на Edge Layer-а.
